归并排序 (Merge)

实现原理

  • 通过分而治之的思想，将两个两个的有序子列进行归并
  • 时间复杂度：稳定在nlogn
  • 优点：稳定的算法
  • 缺点：比较占用内存

实现代码

/**
 * @author Mr.wang

 * function_:归并排序
 */
public class MergeSort &#123;
    public static void main(String[] args)&#123;
        MergeSort m = new MergeSort();
        int[] a = &#123;11,44,23,67,88,65,34,48,9,12&#125;;
        int[] tmp = new int[a.length];
        m.mergeSort(a, 0, a.length - 1, tmp);
        for(int n : a)&#123;
            System.out.print(n + " ");
        &#125;
    &#125;

    public void merge(int[] a, int left, int mid, int right, int[] tmp)&#123;
        int le = left;
        int m = mid + 1;
        int i = 0;
        while (le <= mid && m <= right)&#123;
            if (a[le] < a[m])&#123;
                tmp[i++] = a[le++];
            &#125;
            else&#123;
                tmp[i++] = a[m++];
            &#125;
        &#125;

        //当左右两边仍有剩余时
        while (le <= mid)&#123;
            tmp[i++] = a[le++];
        &#125;
        while (m <= right)&#123;
            tmp[i++] = a[m++];
        &#125;

        //复制回原数组
        for (int j = 0; j < i; j++)&#123;
            a[left+j] = tmp[j];
        &#125;
    &#125;

    public void mergeSort(int[] a,int start, int end,int[] tmp)&#123;
        if(start < end)&#123;
            int mid = (start + end) / 2;
            mergeSort(a, start, mid, tmp);
            mergeSort(a, mid + 1, end, tmp);
            merge(a, start, mid, end,tmp);
        &#125;
        System.out.print("");
    &#125;

Java并发编程基础二

1、多线程并发编程

  • 并发：同一时间段内多个任务同时执行，且执行没有结束
  • 并行：单位时间内多个任务同时执行

2、线程安全问题

  • 只读不会出现问题，至少有一个线程进行修改
  • 通常使用同步锁synchronized进行上锁

3、共享变量的内存可见性问题

  • 变量放在主内存中，线程使用变量时，会将主内存中变量复制到自己的工作空间，然后操作自己工作空间中的变量
  • 通过volatile解决

4、synchronized关键字

  • 当一个线程获取到这个锁后，其他线程必须等待该线程释放锁后才能获取该锁
  • 使用synchronized就是将要使用的变量从工作内存中清除，直接从主存中获取变量
  • 退出synchronized就是将本地内存修改的变量刷新到主存中
  • 使用synchronized会导致上下文切换，增加调度开销
    public synchronized void set（int value）&#123;&#125;

5、volatile关键字

  • 确保对一个变量的更新对其他线程马上可见
  • 从主内存中获取，修改后再刷新回主内存
  • 不保证操作的原子性
  • 适合的使用条件
    • 写入变量值不依赖变量当前值
private volatile int value；

6、原子性操作

  • 含义：执行一系列操作时，这些操作要么全执行，要么全不执行
  • 保证原子性：通过synchronized关键字

7、CAS操作

  • 锁的缺点：当线程没有获取到锁被阻塞时，会导致线程上下文切换和重新调度的开销
  • CAS：Compare and Swap，通过硬件保证原子性
  • boolean compareAndSwapLong（Object obj，long valueOffset，long expect，long update），参数为：对象内存位置、对象中变量的偏移量、变量预期值、新的值。如果对象obj中内存偏移量为valueOffset的变量值为expect，则使用新的值update替换expect
ABA问题
- 线程1准备用CAS将变量的值由A替换为B，在此之前，线程2将变量的值由A替换为C，又由C替换为A，然后线程1执行CAS时发现变量的值仍然为A，所以CAS成功。但实际上这时的现场已经和最初不同了，尽管CAS成功，但可能存在潜藏的问题
- 变量的状态值产生了环形转换     A->B,B->A
- 避免：
    + 1、变量的值只能朝一个方向转换    A->B,B->C
    + 2、JDK中AtomicStampedReference类给每个变量状态值配备了时间戳

#### 线程定义 线程是进程的一个执行路径，一个进程中至少有一个线程，进程中的多个线程共享进程的资源 线程中有程序计数器和栈，程序计数器记录要执行的指令地址，栈存放该线程的局部变量 ***
#### 线程创建和运行 ##### 线程的创建 
 1. 实 现Runnable接口的run()方法
 2. 继承Theread类，重写run方法
 3. 使用FutureTask
三种方式比较
通过继承：方便传参，但不支持多继承
通过Runnable：可以实现多接口，但不方便传参
通过FutureTask：可以拿到任务的返回结果
线程的运行
  • 通过继承Theread类，当调用了start()方法后才真正启动了线程
  • 当run方法执行完毕后，线程就处于终止状态

#### 线程等待和通知 ##### wait() * 调用wait前需获取当前变量监视器锁synchronized * 使用wait后线程进入阻塞，并释放当前变量监视器锁 虚假唤醒：没有使用notify()、notifyAll()，线程也从挂起状态转变为运行状态 ```java //预防虚假唤醒 synchronized(obj){ while (条件不满足){ obj.wait（）； } } ```
wait(long timeout)
在指定时间挂起，超时后则结束并返回;
wait()方法执行的是wait(0);
wait(long timeout,int nanos)
在wait(long timeout)的基础上加入了纳秒级的额外时间
notify()
唤醒一个在该共享变量上调用wait系列方法后被挂起的线程，若存在多个线程等待，则唤醒是随机的
在唤醒前需获得其共享变量的监视器锁
notifyAll()
唤醒所有在该共享变量上调用了wait()方法后而被挂起的线程
只能唤醒调用notifyAll前由于wait而挂起的线程

  • 需要保存现场

x 的平方根

要点

牛顿迭代法
  • 求x的平方根即求f(x)=x^2-a等于0的解，
  • 通过点x的直线于曲线，在无限趋近于x的时候，直线与曲线近似相等
  • 点x处切线交x轴于x0
  • 作x0的垂线交曲线与(x0,f(x0))
  • 通过反复迭代求得x的平方根近似值

要求

实现 int sqrt(int x) 函数。

计算并返回 x 的平方根，其中 x 是非负整数。

由于返回类型是整数，结果只保留整数的部分，小数部分将被舍去。


示例

输入: 4
输出: 2


输入: 8
输出: 2
说明: 8 的平方根是 2.82842..., 
由于返回类型是整数，小数部分将被舍去。

Java 队列api

添加

  • boolean add (E e)
 将指定的元素插入此队列（如果立即可行且不会违反容量限制），在成功时返回 true，如果当前没有可用的空间，则抛出 IllegalStateException

  • boolean offer (E e)
 将指定的元素插入此队列（如果立即可行且不会违反容量限制），当使用有容量限制的队列时，此方法通常要优于 add(E)，后者可能无法插入元素，而只是抛出一个异常
  • 区别
  • add抛出异常
  • offer直接返回false

删除

  • E remove()
 获取并移除此队列的头。此方法与 poll 唯一的不同在于：此队列为空时将抛出一个异常

  • E poll()
获取并移除此队列的头，如果此队列为空，则返回 null

获取

  • E element()
获取，但是不移除此队列的头。此方法与 peek 唯一的不同在于：此队列为空时将抛出一个异常

  • E peek()
获取但不移除此队列的头；如果此队列为空，则返回 null
